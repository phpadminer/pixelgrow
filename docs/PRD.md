## 一、产品概述

### 1.1 产品定位

PixelGrow 是一款面向小学生家庭的**成长型现实RPG系统**，将孩子的真实日常生活映射为像素风虚拟世界中的冒险旅程。通过 AI 驱动的游戏化机制、硬件交互和社交系统，让孩子在"玩"的过程中自然养成良好的学习和生活习惯。

**一句话定义**：孩子人生的数字孪生——现实即游戏，成长即冒险。

### 1.2 核心理念

- **现实即游戏**：虚拟世界时间与现实同步，真实生活事件实时映射到像素世界
- **成长即冒险**：每一天的任务完成、知识学习、技能练习都是角色成长的经验值
- **持续性记录**：不只是一个App，是孩子整个童年的平行世界成长史
- **虚实融合**：通过 ESP32 硬件、3D打印、GPS导航等技术，打通虚拟与现实的边界

### 1.3 目标用户

| 用户角色 | 年龄 | 核心需求 |
|---------|------|---------|
| 小学生（主要用户） | 8-12岁（3-6年级） | 好玩、有成就感、在同学面前有面子 |
| 家长（付费用户） | 30-45岁 | 孩子主动学习、养成好习惯、减少催促 |
| 老师（扩展用户） | - | 班级管理工具、激励学生 |

### 1.4 产品目标

1. 让孩子从"被迫做任务"变成"主动想冒险"
2. 记录孩子完整的成长轨迹，多年后可回看
3. 增强家庭互动，让父母从"监督者"变成"队友"
4. 最终目标：让孩子不再需要这个系统也能自律

## 三、心理学理论基础

### 3.1 自我决定理论（SDT）

游戏化设计必须满足三大内在需求：

- **自主感（Autonomy）**：孩子自己选择做什么任务、设计自己的角色和世界
- **胜任感（Competence）**：AI动态调整难度至"最近发展区"，频繁的"我做到了"时刻
- **归属感（Relatedness）**：宠物是朋友、家庭协作、好友互动

### 3.2 操作性条件反射 → 内在动机转化

**渐进式内化策略**：

| 阶段 | 时间 | 策略 | 目标 |
|------|------|------|------|
| 第1阶段 | 前2周 | 外部奖励建立习惯（每次完成都有金币+动画） | 行为建立 |
| 第2阶段 | 3-6周 | 变比率强化（奖励变随机，有惊喜） | 抗消退 |
| 第3阶段 | 6周后 | 转向叙事奖励、成长回顾、身份认同 | 内在动机 |

**关键**：避免过度合理化效应——不能让孩子只为金币学习。

### 3.3 心流理论

AI实时感知孩子水平，把任务难度调整到"心流通道"（不太难也不太简单）。

### 3.4 费曼学习法

"教分身知识"机制：孩子角色从被动接收者变为主动教授者，理解深度显著提升。

### 3.5 设计红线

| 禁止项 | 原因 |
|--------|------|
| 不做排名 | 10岁自我意识敏感，排名靠后→自我否定 |
| 不惩罚 | 宠物不会饿死/惩罚，只会"休息" |
| 不贴标签 | 不说"你很懒"，只说"你的冒险者今天在休息" |
| AI不替代思考 | 过度使用AI的孩子发散思维下降25% |
| 不过度监控 | 家长看叙事报告，不看原始成绩数据 |

## 四、核心功能模块

### 4.1 3D数字分身系统

#### 4.1.1 角色创建

- 像素/体素风格（我的世界风格），与虚拟世界统一
- 孩子可自定义捏脸：肤色、发型、表情
- 哥哥、妹妹各一个角色
- 家长也可创建角色（参与家庭活动）

#### 4.1.2 多维度成长

角色成长不是一条线，而是多维发展：

| 成长维度 | 对应任务类型 | 外观变化 |
|---------|-------------|---------|
| 智慧 | 学习任务 | 智慧光环、书卷装备、法杖 |
| 生活力 | 生活任务 | 百宝腰包、工具装备 |
| 社交 | 社交/家庭任务 | 友爱光环、守护盾牌 |
| 创造力 | 画画/写作 | 彩虹翅膀、画笔法杖 |
| 艺术 | 音乐/绘画练习 | 专属职业装备 |
| 运气 | 随机/惊喜任务 | 隐藏皮肤、稀有特效 |

#### 4.1.3 专属职业线

- **哥哥 - 音乐骑士线**：见习号手 → 铜管学徒 → 号角战士 → 圆号骑士 → 交响指挥官
- **妹妹 - 创造法师线**：涂鸦学徒 → 色彩魔女 → 造物画师 → 世界设计师 → 创世画神

#### 4.1.4 成长时间轴

- 每个阶段保存角色快照
- 年底生成"年度成长动画"
- 3D打印任意阶段的角色手办

#### 4.1.5 3D打印联动

- 当前形象一键导出STL → 3D打印
- 每学期打印一次 = 书架上的成长记录
- 特殊成就打印纪念版（带底座刻字）

### 4.2 四维任务系统

#### 4.2.1 任务类型

| 类型 | 示例 | 对应属性 | 验证方式 |
|------|------|---------|---------|
| 📚 学习任务 | 完成作业、阅读30分钟、背单词、写作文 | 智慧 | 计时/AI检测 |
| 🏠 生活任务 | 叠衣服、拿快递、收碗筷、整理房间 | 生活力 | 拍照AI识别/NFC/GPS |
| 🤝 社交任务 | 帮妹妹讲题、拿试卷、教爸妈新知识 | 社交 | 家人确认 |
| ⭐ 惊喜任务 | 每天随机刷新的创意任务 | 运气/创造力 | 多种 |

#### 4.2.2 任务验证机制

- **拍照验证**：AI识别（叠好的衣服、整理好的书桌）
- **NFC验证**：碰贴纸打卡（书桌、门口等固定位置）
- **GPS验证**：到达指定地点（拿快递、上兴趣班）
- **计时验证**：手机朝下放置计时（阅读、练琴）
- **家人确认**：轻量级一键确认（帮妹妹讲题）
- **语音/文字**：AI判断内容质量（回答问题）
- **音频检测**：识别乐器练习（圆号）

#### 4.2.3 三层奖励节奏

| 周期 | 奖励类型 | 心理机制 |
|------|---------|---------|
| 每日 | 即时反馈：动画、音效、经验条、每日惊喜 | 多巴胺 |
| 每周 | Boss战、新地图区域、3D打印小徽章 | 成就感 |
| 每月 | 宠物进化仪式、AI生成冒险故事、进化手办 | 身份认同 |

### 4.3 AI知识场景引擎

#### 4.3.1 核心机制：教分身知识

- 虚拟世界中分身遇到问题 → 需要孩子"教"它
- 孩子在教的过程中自然学会知识（费曼学习法）
- 分身不是工具，是"需要帮助的朋友"

#### 4.3.2 课本同步

- 家长设置：年级、教材版本、当前进度
- AI追踪薄弱知识点
- 自动生成与当前学习内容相关的冒险场景

#### 4.3.3 场景示例

| 学科 | 场景 | 玩法 |
|------|------|------|
| 数学 | 商店买剑需要算价格 | 教分身计算 |
| 语文 | 写描写段落才能开门 | 教分身描写技巧 |
| 英语 | 外国NPC说英文求助 | 教分身翻译+回复 |
| 科学 | 花园种子不发芽 | 教分身科学常识 |
| 综合 | 隐藏Boss需要多学科能力 | 综合挑战 |

#### 4.3.4 知识树系统

- 每个学科一个分支，可视化展示
- 知识树越茂盛→分身能去的地方越多
- 知识树可以3D打印成实体摆件

#### 4.3.5 难度动态调整

- 连续3次轻松完成 → 升级难度
- 连续2次失败 → 降低难度 + 鼓励 + 提示
- 始终保持在"最近发展区"

### 4.4 作文→动漫系统

#### 4.4.1 核心流程

```
孩子写作文（打字/手写拍照OCR）
→ 大模型分析：提取3-5个关键场景
→ 为每个场景生成画面描述prompt
→ AI角色一致性设定（孩子的3D分身当主角）
→ 文生视频API（可灵/即梦）逐场景生成
→ TTS配音（用孩子作文当旁白）+ BGM
→ 自动剪辑合成 → 30秒-2分钟动漫短片
```

#### 4.4.2 教育价值

| 传统痛点 | 解决方式 | 心理学原理 |
|----------|---------|-----------|
| 写了没人看 | 变成动画，全家看，可分享 | 观众效应 |
| 不知道写什么 | AI画面激发想象 | 具象化脚手架 |
| 不想修改 | 改一句→画面变→好玩 | 游戏化迭代 |
| 写长了太累 | 想要更多场景→自然写更长 | 内在动机驱动 |

#### 4.4.3 积分联动

- 写作文 → 额外大量积分
- 积分解锁：更多动漫风格、更长时长、特效、自定义BGM
- 可以把宠物加入动画当配角

#### 4.4.4 兄妹协作

- 哥哥写故事 + 妹妹画插图 → AI合成
- 哥哥文字当旁白 + 妹妹画风当美术
- 音画同步：哥哥吹旋律→妹妹听感觉画画→合成MV

### 4.5 哥哥·圆号系统

#### 4.5.1 练琴检测

- 手机/ESP32麦克风检测圆号练习
- AI识别：确认在吹圆号（不是瞎吹）
- 检测练习时长 + 基础音准分析

#### 4.5.2 练琴→游戏

| 练琴行为 | 游戏反馈 |
|---------|---------|
| 开始练琴 | 分身进入"修炼模式"动画 |
| 每10分钟 | 获得一个能量单位 |
| 音准好 | "完美充能！双倍经验" |
| 有进步 | "你的音准比昨天进步了3%" |
| 练会新曲 | 曲目变成虚拟世界某区域的BGM |

#### 4.5.3 音乐Boss战

- 吹指定音阶→释放"音波攻击"
- 渐进难度：音阶→简单旋律→完整乐句
- 集齐乐谱碎片→解锁"演奏厅"区域

#### 4.5.4 AI圆号教练

- 练完一段→AI分析音准、节奏
- 给出具体改进建议
- 分身软性鼓励："再来一遍好不好？"

### 4.6 妹妹·画画系统

#### 4.6.1 画什么出现什么

- 妹妹画一幅画 → 拍照/平板直接画
- AI识别+像素化→出现在虚拟世界里
- 画花→种在花园，画猫→变NPC，画房子→变建筑，画衣服→分身穿上

#### 4.6.2 AI绘画伙伴

- 分身陪画画，提出需求（"我想要一只小狗"）
- 引导观察细节（"好像少了尾巴？"）
- 不评判好坏，陪伴式引导

#### 4.6.3 画作→3D打印

- AI将2D画作转成3D模型
- 打印出来："这个小狗是我画的，现在变成真的了！"

### 4.7 GPS现实任务系统

#### 4.7.1 核心体验

家长发布出门任务→孩子接受→进入地图导航模式→分身在像素化地图上实时移动→路上触发随机事件（金币/NPC问答/宠物）→到达目的地→开启宝箱→返程额外奖励

#### 4.7.2 幻想化映射

**原则：真实的GPS坐标和距离 + 幻想化的名字和画面**

| 现实地点 | 虚拟化名称 |
|---------|-----------|
| 家 | 冒险者的城堡 |
| 学校 | 智慧学院 |
| 超市 | 炼金商店 |
| 快递站 | 宝藏洞穴 |
| 公园 | 精灵森林 |
| 奶奶家 | 长老的塔 |
| 兴趣班 | 修炼圣地 |

- 孩子可以自己给新地点命名
- 同一地点每次去有不同剧情（AI生成）
- 迷雾系统：只有去过的地方才"亮"

#### 4.7.3 路径随机事件

每走50-100米可能触发：
- 🪙 金币掉落
- ❓ 知识问答（与当前学科进度相关）
- 🐾 野生宠物（回答对可收服）
- 📸 拍照任务
- 🌟 路径成就

#### 4.7.4 安全机制（最高优先级）

- **电子围栏**：超出范围→家长警报→游戏暂停
- **任务只能家长发布**：孩子不能自己创建出门任务
- **实时位置共享**：任务期间家长端实时看到位置
- **过马路提醒**：接近路口→弹出安全提醒→暂停游戏
- **沉迷防护**：导航模式不需一直看手机，到事件点手环震动再看

### 4.8 像素虚拟世界

#### 4.8.1 世界构成

- 时间与现实同步（日夜、四季）
- 天气与现实同步（天气API）
- 以家为中心，随探索不断扩展
- 迷雾系统：未探索区域被遮挡

#### 4.8.2 区域类型

| 区域 | 解锁条件 |
|------|---------|
| 冒险者小镇 | 初始区域 |
| 知识学院 | 学习任务Lv.5 |
| 精灵森林 | 实际去过公园 |
| 数字迷宫 | 数学知识树Lv.10 |
| 故事王国 | 语文知识树Lv.10 |
| 环球港口 | 英语知识树Lv.10 |
| 演奏厅 | 音乐Lv.15（哥哥专属） |
| 创造工坊 | 绘画Lv.15（妹妹专属） |
| 智者之塔 | 全属性Lv.10 |

#### 4.8.3 剧情系统

**主线剧情**（按季度推进）：
- 第一季（1-3月）：新手村的日子
- 第二季（4-6月）：探索者的旅途
- 第三季（7-9月）：守护者的崛起
- 第四季（10-12月）：光明回归
- 一年一个完整故事弧

**支线剧情**（AI根据真实行为动态生成）：
- 孩子数学做得好→数字王国求助任务
- 孩子没运动→温和的"分身需要力量"剧情
- 兄妹吵架→"双子塔裂缝"合作修复任务

**节日/生日特别剧情**：
- 春节：年兽Boss、限定皮肤
- 生日：全NPC庆生、生日限定宠物
- 考试周：用游戏缓解焦虑

### 4.9 AI宠物伴侣系统

#### 4.9.1 宠物角色

- 是"朋友"不是"工具"
- 有性格、有情感表达
- 不说"你今天没完成任务"，说"你能帮我吗？"
- 永远不会死亡，只会"休息"

#### 4.9.2 宠物功能

- 日常陪伴：显示在ESP32宠物机上
- 任务引导：用故事包装任务
- 知识求助：宠物向孩子"求教"
- 情感支持：鼓励、安慰、庆祝
- 收集系统：路上可以遇到野生宠物

#### 4.9.3 宠物进化

- 按等级进化外观
- 不同喂养方式→不同进化方向
- 进化仪式（全息盒/投影展示）
- 可以3D打印每个进化阶段

### 4.10 社交系统

#### 4.10.1 分层设计

| 层级 | 范围 | 开启条件 |
|------|------|---------|
| Lv.1 家庭社交 | 家庭内部 | 默认开启 |
| Lv.2 好友社交 | 同学/朋友 | 家长审批 |
| Lv.3 班级社交 | 全班 | 老师+家长管理 |

#### 4.10.2 家庭社交

- 家庭任务板：任何家人可发布/接受任务
- 周末Boss战：全家参与，各有角色
- 家庭信箱：通过宠物传递鼓励信
- 家庭成就墙：全家合力达成目标

#### 4.10.3 好友社交

- **加好友**：线下交换邀请码 + 双方家长同意
- **能做的**：看角色装备、参观世界、互送礼物、协作副本
- **不能做的**：看真实位置、文字聊天、看家庭信息
- **朋友圈**：只能发角色截图/成就/画作，只能点赞（预设正面表情），无评论

#### 4.10.4 班级社交

- 班级世界共建：每人贡献一个区域
- 多维度之星：进步之星、创造之星、助人之星（轮换制）
- 班级故事接龙→AI合成班级动漫
- 班VS班挑战（良性竞争）

#### 4.10.5 社交货币

所有社交炫耀元素都指向"我努力了/我有才华"，不是"我充钱了"：
- 酷炫角色装备 = 长期坚持
- 稀有宠物 = 特殊成就
- 自创BGM = 真实技能
- 3D打印手办 = 带到学校的实体社交货币

#### 4.10.6 安全设计

- 无文字聊天→无法发攻击性言语
- 只能正面互动（点赞/送礼/合作）
- 无法破坏别人的世界
- 拒绝好友请求→不通知被拒者
- 社交每日限时30分钟
- 21:00后自动关闭
- 学校时间关闭社交功能

---

### 4.11 家长端

#### 4.11.1 核心原则

家长是**支持者**，不是**监督者**。

#### 4.11.2 功能

| 功能 | 说明 |
|------|------|
| 发布任务 | 选择类型、目的地、奖励 |
| 成长叙事 | 看到的是"故事"不是"成绩单" |
| 安全监控 | GPS任务时实时位置 |
| 确认任务 | 一键确认孩子完成 |
| 兑换管理 | 设定金币→现实奖励兑换表 |
| 家庭信箱 | 通过宠物给孩子写鼓励信 |
| 社交管控 | 审批好友、设定限时 |

#### 4.11.3 家长不能看到的

- ❌ 原始错题和分数
- ❌ 孩子的聊天内容
- ✅ 只看"成长叙事报告"：
  - "这周你的探索范围增加了3个区域"
  - "你的宠物学会了2个新技能"
  - "建议晚上聊聊他在数学冒险中遇到了什么"

## 五、硬件生态

### 5.1 ESP32 宠物机（核心设备）

| 组件 | 型号 | 功能 | 价格 |
|------|------|------|------|
| 主控 | ESP32-S3 Mini | WiFi+蓝牙+GPIO | ~35元 |
| 屏幕 | 1.69寸 ST7789 TFT | 显示像素宠物 | ~25元 |
| NFC | RC522 | 打卡识别 | ~8元 |
| 蜂鸣器 | 无源 | 音效 | ~2元 |
| LED | WS2812 x3 | 状态灯 | ~3元 |
| 电池 | 3.7V 600mAh | 供电 | ~10元 |
| 充电 | TP4056 | USB-C充电 | ~3元 |
| 外壳 | 3D打印 | 我的世界风格 | ~10元 |
| **合计** | | | **~96元** |

### 5.2 NFC 打卡贴纸

- 贴在书桌、门口、钢琴等固定位置
- 碰一下=打卡，仪式感强
- 成本：几毛钱一张

### 5.3 扭蛋机（积分兑换）

- ESP32 + SG90舵机
- 积分够→输入兑换码→扭蛋掉出
- 里面是3D打印的随机宠物/装备
- 仪式感极强

### 5.4 魔法底座（Toniebox式）

- 3D打印手办（内嵌NFC）
- 放到底座上→屏幕显示该宠物状态/故事
- 两只宠物同时放→触发合成/战斗

### 5.5 全息棱镜盒

- 手机+亚克力棱镜（~20元）
- 宠物"悬浮"在盒子里
- 进化仪式在全息盒中展示

### 5.6 自研定位手环（进阶）

- ESP32-S3 + GPS模块(ATGM336H) + 加速度计
- 蓝牙同步手机→上传云端
- NFC+震动反馈
- 成本~80-120元

### 5.7 3D打印系统

- 推荐打印机：Bambu Lab A1 mini（~1500元）
- 可打印物品：
  - 角色手办（每学期一个=成长记录）
  - 宠物进化系列
  - 成就奖杯/徽章
  - 妹妹的画作→3D模型
  - 迷你圆号模型
  - 扭蛋机外壳
  - 宠物机外壳

### 6.2 技术栈

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| 孩子端 | 微信小程序 + Phaser.js/PixiJS | 无需下载，像素游戏渲染 |
| 家长端 | 微信小程序 | 同一生态 |
| 后端 | Node.js (NestJS) 或 Python (FastAPI) | 快速开发 |
| 数据库 | MySQL + Redis + MongoDB | 结构化+缓存+轨迹 |
| 地图 | 腾讯地图SDK/高德地图 | 小程序原生支持 |
| AI-LLM | 通义千问/豆包 API | 场景生成/剧情/分析 |
| AI-视频 | 可灵/即梦 API | 作文→动漫 |
| AI-TTS | 火山引擎 TTS | 配音 |
| AI-音频 | 基础频率分析 | 圆号音准检测 |
| 存储 | 阿里云OSS/腾讯COS | 视频/图片/模型 |
| 硬件 | ESP32-S3 + Arduino/PlatformIO | 宠物机/手环/扭蛋机 |
| 3D建模 | Three.js / MagicaVoxel | 体素角色生成 |

### 6.3 地图实现

- 微信小程序 `map` 组件 + `MapContext` API
- `wx.getLocation` 实时定位
- `markers` 自定义像素风图标（金币/NPC/宝箱/分身）
- `MapContext.moveAlong` 分身沿路径移动动画
- `polyline` 导航路径绘制
- 腾讯/高德地图插件做路径规划

```
User {
  id, family_id, role (child/parent)

}
```

### 7.3 角色

```
Character {
  id, user_id
  level, exp
  attributes: { wisdom, life_skill, social, creativity, art, luck }
  equipment: { head, body, hand, back, pet_companion }
  profession: { type, level, skills }
  appearance: { height, skin, hair, expression }
  snapshots: [{ date, model_data }]
}
```

### 7.4 任务

```
Task {
  id, family_id
  type: (study/life/social/surprise)
  title, description
  target_user_id
  verification_type: (photo/nfc/gps/timer/confirm/voice)
  gps_data: { start, end, waypoints, events }
  rewards: { exp, coins, items }
  status, completed_at
}
```

### 7.5 知识场景

```
KnowledgeScene {
  id, date
  theme, location_type
  subjects: [{ subject, topic, difficulty }]
  scenes: [{ description, prompt, quiz, reward }]
  boss: { description, requirements, reward }
}
```

### 7.6 虚拟世界

```
World {
  id, owner_id
  regions: [{ name, fantasy_name, unlocked, real_location? }]
  buildings: [{ type, created_by, artwork? }]
  npcs: [NPC]
  bgm_tracks: [{ track, recorded_by }]
  weather: synced_with_real
  time: synced_with_real
  exploration_rate: percentage
}
```
